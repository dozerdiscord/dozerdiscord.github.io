Intro

Howdy! This is the quickstart guide for Dozer. This will walk you through how to get the best experience from Dozer on your Discord server. Not all of these steps are required, but these are recommended for the best experience.

Webhook setup

Dozer has an official Twitter account that is used to notify users of outages and other important information. If you would like to have this feed relayed to a channel in your server, please create a webhook in the channel you would like the feed to go to and send the webhook URL to tweirtx (Travis from FRC team 5052) You can also follow the Twitter account directly, which is @DozerDiscord.

Join/leave logs

If you want Dozer to send embeds when people join or leave, all you have to do is give Dozer send permissions in the right channel and run &memberlogconfig (channel) where (channel) is a #channel-name.

Message edit/deletion logs

To set up logs of when someone edits/deletes a message (typically used for moderation purposes), run &messagelogconfig (channel) where (channel) is a #channel-name

Giveme roles

To add a role to the list of roles people can give themselves, simply run &giveme add (role name). To remove, do &giveme removefromlist (role name).

Modlogs

Want to use this bot for moderation and have it log all actions to a channel? You can do so with &modlogconfig #channel-name

Member role

This is necessary for some things to work correctly. This will tell Dozer what role is your guild's role for verified members. To do this, do &memberconfig (role)

Voicebinds

You can configure certain channels to only appear to a member when they are in a certain voice chat. To do this, do &voicebind "Voice channel name" #text-channel and to unbind, do &voiceunbind "Voice channel"

New member rule acceptance

Dozer has a feature where you can require a certain phrase to be sent by a user before they are given access to the server. To do this, run &nmconfig #channel-name (members role name) (Message you want new users to send.

Link Scrubbing

You can restrict the sending of links at all unless a user has a certain role. To set this up, do &linkscrubconfig (role users must have to send).
